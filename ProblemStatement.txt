Market Research Intelligence Assistant

Business Problem

Product and GTM teams struggle to stay current on competitor activity and market trends because
relevant information is scattered across blogs, websites, announcements, and articles. Build a lightweight
application that helps a user collect, analyze, and summarize market intelligence from multiple public
sources.

Assignment Description
Create a web application that allows a user to:
1. Provide a list of competitor names or topics
2. Provide one or more source URLs (blogs, announcement pages, articles)
3. Generate a structured market intelligence summary, including:
o Key themes or trends
o Notable competitor activities
o Source references for each insight
o Check for hallucination using LLM-as-a-judge to verify the claims in the output report

Expected Scope (Baseline)
 Input form for topics / competitors + URLs
 Fetch and process content from provided sources
 Implement a research phase on the provided topics and resouces
 AI-generated summary grouped by themes
 Source traceability (which insight came from which URL)
 Implement authentication
 Simple persistence (store previous runs or links to analyze)
 Deploy the application using a cloud platform (Azure preferred)
Stretch Goals (Optional)
 Change detection (“what’s new since last run”)
 Simple monitoring

Success Criteria
 The output is useful, structured, and readable
 Insights are clearly grounded in sourcesgive th
 The system handles at least 3–5 URLs reliably
 The app can be accessed via a public URL
Expected Deliverables
 Live hosted URL
 Source code repository (GitHub or equivalent)
 README with:
o Problem statement
o Architecture &amp; tech stack
o Local build &amp; run instructions
o AI tools / models / libraries used (with citations)
 Written explanation of design decisions
